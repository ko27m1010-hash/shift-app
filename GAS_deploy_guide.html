📊 STEP 1：Googleスプレッドシートを作成
1

Google ドライブを開く

パソコンのブラウザで drive.google.com を開きます。

drive.google.com
🗂️ マイドライブ画面が表示されます
左上の ＋ 新規 ボタンをクリック

「＋ 新規」→「Googleスプレッドシート」→「空白のスプレッドシート」を選択します。

💡 ヒント

既存のスプレッドシートを使う場合もOKです。シフトデータ用に新しいシートを1枚追加してください。

2

スプレッドシートに名前をつける

左上の「無題のスプレッドシート」をクリックして、わかりやすい名前に変更します。

📋
例）「シフト管理_農業パート」
💡 おすすめの名前

「シフト管理_農業パート_2026」など、年を入れておくと後で探しやすいです。

📝 STEP 2：Apps Scriptにスクリプトを設定
3

Apps Script エディタを開く

スプレッドシート上部のメニューから操作します。

  1. メニューバーの「拡張機能」をクリック
  2. 「Apps Script」をクリック
script.google.com/...
🖥️ Apps Scriptエディタが別タブで開きます
「function myFunction() { }」というコードが最初から書いてあります
4

スクリプトをコピー＆ペースト

配布ファイルに含まれる「GAS_Webhook受信スクリプト.js」をテキストエディタで開きます。

  1. ファイル全体を選択（Ctrl+A / Command+A）してコピー
  2. Apps Scriptエディタの既存コードを全て削除する
  3. コピーしたコードを貼り付け（Ctrl+V / Command+V）
⚠️ 重要

既存の「function myFunction() { }」は必ず削除してから貼り付けてください。2重になるとエラーになります。

// このようなコードが貼り付けられればOK const SHEET_NAME = 'シフト希望'; function doPost(e) { const data = JSON.parse(e.postData.contents); // スプレッドシートに保存する処理 ... }
5

スクリプトを保存する

フロッピーディスクアイコン（💾）または Ctrl+S（Command+S） で保存します。

✅ 保存すると画面上部に「最終保存：〇〇分前」と表示されます。

🚀 STEP 3：ウェブアプリとしてデプロイ
6

デプロイメニューを開く

Apps Scriptエディタの右上にある「デプロイ」ボタンをクリックします。

script.google.com/macros/...
デプロイ ▼
ドロップダウンメニューが表示されます↓
➕ 新しいデプロイ
デプロイを管理

「➕ 新しいデプロイ」をクリックします。

7

デプロイタイプを設定する

「新しいデプロイ」ダイアログが開きます。以下の通りに設定してください。

⚙️
種類の選択（歯車アイコン）をクリック
→「ウェブアプリ」を選択
  1. 「説明」欄に「シフト管理アプリ」と入力（任意）
  2. 「次のユーザーとして実行」→ 「自分（あなたのメールアドレス）」を選択
  3. 「アクセスできるユーザー」→ 「全員」を選択 ⭐
⚠️ 「全員」を必ず選択してください

「全員」にしないとパートスタッフがフォームを送信できません。スプレッドシートに書き込む権限は自分のアカウントで実行されるため、外部に情報は漏れません。

8

デプロイを実行・URLをコピー

「デプロイ」ボタンをクリックします。

🔐 初回は権限の確認が必要です

「アクセスを承認」→「Googleアカウントを選択」→「詳細」→「(スクリプト名)（安全でないページ）に移動」→「許可」の順でクリックしてください。

🎉
デプロイ成功！ウェブアプリのURLが表示されます
// このような形式のURLが表示されます https://script.google.com/macros/s/AAAA...ZZZZ/exec

このURLをコピーしてください（後のステップで使います）。

✅ 「ウェブアプリのURL」の横にある「コピー」ボタンを押すと確実です。

📱 STEP 4：アプリにURLを設定
9

アプリの管理者設定でURLを入力

シフト管理アプリをスマホまたはPCで開きます。

  1. 下部タブの「⚙️ 管理」をタップ（PINを入力：初期値 1234）
  2. 「🔗 スプレッドシート連携（GAS URL）」セクションを見つける
  3. コピーしたURLを貼り付ける
  4. 「保存」ボタンをタップ

✅ 「URLを保存しました。スタッフ全員が使えるようになりました！」と表示されれば設定完了です。

⚠️ 注意：管理者の端末で1回だけ設定します

この設定は管理者のブラウザに保存されます。スタッフの端末では設定不要です。スタッフはそのままアプリを開いてシフトを入力するだけです。

10

動作確認をする

テスト送信をして、スプレッドシートに記録されるか確認します。

  1. アプリの「📝 入力」タブでテストデータを入力・送信
  2. Googleスプレッドシートを開く
  3. 「シフト希望」シートにデータが追加されていればOK！
📊
A列: 送信日時
B列: 名前
C列: 対象月
D列: 出勤パターン
E列: 出勤日
F列: 時間帯
💡 データが記録されない場合

① GASのURLが正しいか確認
② デプロイが「全員」になっているか確認
③ Apps Scriptで「実行」→「doPost」で手動テストを試す

❓ よくある質問
Q

スクリプトを変更したらURLが変わりますか？

はい。スクリプトを変更した場合は「デプロイを管理」→「バージョンを編集」→「新しいバージョン」として再デプロイしてください。

再デプロイ後はURLが変わることがあるので、アプリの管理者設定で再度URLを更新してください。

Q

スプレッドシートが共有されても大丈夫？

GASのウェブアプリURLを知っている人だけがデータを送信できます。スプレッドシート自体を誰かと共有するかどうかは管理者が設定でき、デフォルトでは管理者のみアクセスできます。

Q

無料で使えますか？

はい。Google Apps ScriptはGoogleアカウントがあれば無料で使えます。1日の実行回数の上限（6分/回など）がありますが、シフト管理程度の用途では上限に達することはほぼありません。
